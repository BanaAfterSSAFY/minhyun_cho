# [백준 13023] ABCDE: 깊이 우선 탐색(DFS)으로 친구 관계 찾기

## 1. 문제 요약 및 풀이 전략

### 문제 요약

`N`명의 사람과 `M`개의 친구 관계가 주어집니다. 이 친구 관계는 양방향입니다. 우리는 "A는 B와 친구, B는 C와 친구, C는 D와 친구, D는 E와 친구"인 5명의 친구 관계(즉, A-B-C-D-E로 이어지는 길이 4의 경로)가 존재하는지 여부를 찾아야 합니다. 존재한다면 `1`을, 존재하지 않는다면 `0`을 출력하는 문제입니다.

**예시**:
*   A-B
*   B-C
*   C-D
*   D-E
이런 관계가 하나라도 존재하면 `1`입니다.

### 풀이 전략

이 문제는 전형적인 **그래프 탐색** 문제로 볼 수 있습니다. `N`명의 사람을 정점(Node)으로, `M`개의 친구 관계를 간선(Edge)으로 하는 그래프를 생각할 수 있습니다. 우리가 찾아야 하는 것은 **정점 5개로 이루어진 단순 경로(Simple Path)**, 즉 길이가 4인 경로입니다.

이를 해결하기 위한 가장 적합한 알고리즘은 **깊이 우선 탐색(DFS)**입니다.

1.  **그래프 표현**: 친구 관계는 양방향이므로, 인접 리스트(Adjacency List)를 사용하여 각 정점에 연결된 정점들을 저장합니다. `graph.get(u).add(v);`와 `graph.get(v).add(u);`를 통해 양방향 관계를 표현합니다.
2.  **모든 정점에서 시작**: A-B-C-D-E 경로의 시작점이 될 수 있는 정점은 모든 `N`개의 정점 중 하나일 수 있습니다. 따라서 `main` 함수에서는 `0`부터 `N-1`까지의 모든 정점에 대해 DFS를 시작합니다.
3.  **DFS 탐색**:
    *   `dfs(int count, int target)` 함수는 현재까지 탐색한 깊이(`count`)와 현재 방문 중인 정점(`target`)을 인자로 받습니다.
    *   **종료 조건**: `count`가 4가 되면 (이는 5개의 정점, 즉 A-B-C-D-E 관계를 찾았음을 의미) `found` 전역 변수를 `true`로 설정하고 재귀 호출을 종료합니다.
    *   **방문 체크**: `isVisited` 배열을 사용하여 이미 방문한 정점은 다시 방문하지 않도록 합니다. 이는 "단순 경로"를 찾기 위함입니다 (같은 사람을 두 번 이상 거치지 않음).
    *   **백트래킹**: 한 경로를 탐색하다가 막다른 길에 도달하거나 목표를 찾지 못하고 돌아올 때는, 해당 정점을 `isVisited[target] = false;`로 설정하여 다음 DFS 경로에서 다시 방문할 수 있도록 해야 합니다. 이 백트래킹 과정이 매우 중요합니다.
4.  **조기 종료**: `found` 변수가 `true`가 되면, 더 이상 탐색을 진행할 필요가 없습니다. `main` 함수와 `dfs` 함수 내에서 `if (found) return;` 조건을 추가하여 불필요한 탐색을 줄입니다.

## 2. 시간 복잡도와 공간 복잡도 분석

### 시간 복잡도

*   **그래프 초기화**: `N`개의 `ArrayList`를 생성하는 데 `O(N)` 시간이 소요됩니다.
*   **간선 입력**: `M`개의 간선을 `ArrayList`에 추가하는 데 `O(M)` 시간이 소요됩니다. 양방향이므로 각 간선당 두 번의 추가 연산이 있습니다.
*   **DFS 탐색**:
    *   `main` 함수에서 `N`개의 모든 정점을 시작점으로 하여 DFS를 시도합니다. `for (int i = 0; i < N; i++)`.
    *   각 DFS 호출은 최대 깊이 4까지 탐색합니다. `isVisited` 배열로 중복 방문을 방지하므로 단순 경로만을 탐색합니다.
    *   일반적인 DFS의 시간 복잡도는 `O(V+E)`입니다. 여기서는 `N`개의 시작점에서 각각 최대 깊이 4까지의 탐색을 수행하므로, 최악의 경우 **`O(N * (N+M))`**이 될 수 있습니다.
        *   `N`은 최대 2000, `M`은 최대 2000이므로, `2000 * (2000 + 2000) = 2000 * 4000 = 8,000,000` (8백만) 연산입니다. 이 정도면 주어진 시간 제한(보통 1~2초) 내에 충분히 해결 가능합니다. `found` 플래그를 통한 조기 종료 덕분에 실제로는 훨씬 적은 시간이 소요될 수 있습니다.

따라서 전체 시간 복잡도는 **`O(N * (N+M))`** 입니다.

### 공간 복잡도

*   **`graph` (인접 리스트)**: `N`개의 `ArrayList` 객체를 저장하며, 총 `2M`개의 정수(간선 정보)를 저장합니다. 따라서 `O(N+M)`의 공간이 필요합니다.
*   **`isVisited` (방문 배열)**: `N`개의 boolean 값을 저장하므로 `O(N)`의 공간이 필요합니다.
*   **DFS 재귀 스택**: 최대 깊이가 4로 고정되어 있으므로, 재귀 스택의 깊이는 최대 5입니다. 이는 상수 공간 **`O(1)`**로 볼 수 있습니다.

따라서 전체 공간 복잡도는 **`O(N+M)`** 입니다.

## 3. 코드 개선점 및 주의할 예외 케이스 (Edge Case)

### 코드 개선점

1.  **전역 변수 `found` 사용 지양**: `found`와 같은 전역 변수는 코드의 가독성을 해치고 디버깅을 어렵게 만들 수 있습니다. DFS 함수가 `boolean` 값을 반환하도록 변경하여 목표를 찾았는지 여부를 호출자에게 알리는 것이 더 좋은 설계입니다.
    ```java
    // 개선 전:
    // static boolean found = false;
    // private static void dfs(int count, int target) {
    //     if (count == 4) { found = true; return; }
    //     ...
    //     if (found) return;
    //     ...
    // }

    // 개선 후:
    private static boolean dfs(int count, int target) {
        if (count == 4) {
            return true; // 찾았음을 반환
        }

        isVisited[target] = true;
        for (int next : graph.get(target)) {
            if (!isVisited[next]) {
                if (dfs(count + 1, next)) { // 재귀 호출 결과 확인
                    return true; // 찾았으면 바로 true 반환
                }
            }
        }
        isVisited[target] = false; // 백트래킹
        return false; // 못 찾았으면 false 반환
    }

    // main 함수에서 사용:
    // for (int i = 0; i < N; i++) {
    //     if (dfs(0, i)) { // DFS 결과 바로 사용
    //         found = true; // (또는 System.out.println("1"); return;)
    //         break;
    //     }
    // }
    ```
    이렇게 변경하면 `found` 전역 변수 없이도 같은 로직을 구현할 수 있으며, 함수 간의 의존성이 낮아집니다. (현재 코드에서는 `main` 함수의 `found`와 `dfs` 함수의 `found`가 함께 쓰이므로 조기 종료는 잘 되지만, 함수형 관점에서는 위와 같은 개선이 더 바람직합니다.)

2.  **매직 넘버 상수화**: `if (count == 4)` 에서 `4`는 "길이 4의 경로"를 의미하는 중요한 숫자입니다. 이를 상수로 선언하여 코드의 의미를 명확히 할 수 있습니다.
    ```java
    // static final int TARGET_PATH_LENGTH = 4;
    // ...
    // if (count == TARGET_PATH_LENGTH) { ... }
    ```

### 주의할 예외 케이스 (Edge Case)

1.  **N < 5 인 경우**: 문제에서 요구하는 A-B-C-D-E 관계는 최소 5명의 사람이 필요합니다. 만약 `N`이 5 미만이라면, 이 관계는 절대로 존재할 수 없습니다. 현재 문제의 제약 조건 `N (50 <= N <= 2000)`에 따르면 이 경우는 발생하지 않습니다. 하지만 일반적인 그래프 문제에서는 이러한 최소 정점 수를 고려해야 합니다. 현재 코드는 `N < 5`인 경우 `found`가 절대 `true`가 될 수 없으므로 `0`을 잘 출력할 것입니다.

2.  **그래프가 비어있는 경우 (M = 0)**: 친구 관계가 하나도 없는 경우입니다. 당연히 길이 4의 경로는 존재할 수 없습니다. 현재 코드는 `M=0`이면 `graph`에 아무런 간선도 추가되지 않고, `dfs` 내부의 `for (int next : graph.get(target))` 루프가 실행되지 않으므로 `found`는 `false`로 유지되고 `0`을 올바르게 출력합니다.

3.  **Disconnected Graph (단절된 그래프)**: 그래프가 여러 개의 연결 컴포넌트로 나뉘어져 있을 수 있습니다. 예를 들어, 1-2-3-4-5와 6-7-8-9-10 두 개의 그룹이 전혀 연결되어 있지 않을 수 있습니다. 현재 코드는 `main` 함수에서 `N`개의 모든 정점을 시작점으로 DFS를 시도하기 때문에, 어떤 연결 컴포넌트에서라도 목표 경로가 존재한다면 찾아낼 수 있습니다. 이 부분은 잘 처리되어 있습니다.

## 결론

백준 13023번 "ABCDE" 문제는 단순한 그래프 탐색을 넘어, DFS를 이용한 특정 길이의 단순 경로 찾기라는 흥미로운 주제를 다룹니다. `isVisited` 배열을 이용한 중복 방문 방지, 그리고 `isVisited[target] = false;`를 통한 백트래킹은 DFS를 사용하여 모든 가능한 경로를 효율적으로 탐색하는 데 필수적인 기법입니다. 또한, `found` 플래그를 이용한 조기 종료는 불필요한 연산을 줄여 시간 효율성을 높이는 중요한 최적화 전략입니다.
